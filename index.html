üîí Security & Best Practices

‚ö†Ô∏è Important Security Considerations

  • MCP has full access to your Back4App account
  • AI can create, modify, and delete apps and data
  • Always test on a dedicated test account first
  • Never share configuration files with access tokens
  • Review all AI-generated database schemas and cloud functions

‚úÖ Recommended Practices

  • Use separate access tokens for MCP
  • Review all AI-generated code before production deployment
  • Enable web hosting for easy deployment visualization
  • Keep Node.js updated (v16 or higher)
  • Implement proper Class Level Permissions (CLPs) for data security
  • Use version control for your project files
  • Test authentication flows thoroughly
  • Monitor application performance and user metrics

üõ†Ô∏è Troubleshooting

Common Issues & Solutions

‚ùå Connection Failures
  • Verify account key is valid and correctly entered
  • Check Node.js version (v16+ required)
  • Ensure npx command works correctly
  • Restart your IDE after configuration changes
‚ùå Server Not Responding
  • Restart your IDE/LLM client
  • Check MCP server status in your tool
  • Verify network connectivity
  • Clear npm cache: npm cache clean --force
‚ùå Permission Errors
  • Ensure account key has necessary permissions
  • Check if account key is expired
  • Create a new account key if needed
  • Verify app selection in Back4App dashboard
‚ùå Deployment Issues
  • Check file paths and structure
  • Verify web hosting is enabled for your app
  • Ensure all required files are included
  • Check for JavaScript errors in browser console
